## 规范

### 为什么要规范

跟性能无关 - 效率有关  
跟功能无关 - 减少团队 gap  
跟结果无关 - 保留最佳实践  

### 易读
我们写出来的代码要去执行，但是更重要是要被维护，例如该什么时候空格，这个对于保持易读性来说非常重要

### css 规范
css 将属性分3类：布局类，内容类，修饰类

### js 规范
- 变量: 小驼峰(myName)、区分单复数(names 与 name 的区别)
- 对象: 大驼峰(Person)、单数
- 常量: 大写
- 函数: 小驼峰、语义化
- 布尔值: 小驼峰(is,has,can)
- 私有属性: 小驼峰

### 常用规范
- 中文英文或中文数字之间是要有空格的
- 命名要非常注意，这个可以直接看出编程修养的

### 规范管理
- stylelint
- ESLint
- editorconfig（编辑器支持的，统一团队配置）
